# Plan: CMS Integration for Content Writing

## Goal
Give Emilio a user-friendly writing experience (WYSIWYG editor, image drag-and-drop, formatting toolbar) instead of writing raw Markdown files for the Astro blog.

## Recommendation: Keystatic CMS

Add Keystatic, a free git-based CMS that gives the site an admin panel at /keystatic with a rich text editor, image uploads, and one-click publishing.

## What Changes

1. Install @keystatic/core and @keystatic/astro packages
2. Create keystatic.config.ts defining the blog collection (matches existing content schema)
3. Add a /keystatic admin route page
4. Update astro.config.mjs to include Keystatic integration
5. Configure image storage in the repo (public/blog-images/)

## What Stays the Same

- Content still lives as Markdown in src/content/blog/
- Netlify auto-deploys on push to main (unchanged)
- Blog listing page and post template work as-is
- No new external services or databases
- No cost (free, open source)

## Writing Experience

- Rich text editor with toolbar (bold, italic, headings, lists, links, code blocks)
- Drag-and-drop image upload
- Preview pane
- Form fields for title, description, date, author, tags
- Publish button commits to GitHub automatically
- Login via GitHub OAuth (one-time setup)

## Implementation Steps

1. Install dependencies
2. Create keystatic.config.ts
3. Add admin route page
4. Update astro.config.mjs
5. Update content schema to add tags field
6. Test locally with a sample post
7. Deploy to Netlify
8. Update context docs with the CMS decision
